# Product

## Register

product

## Users

Reverie is primarily used in a hackathon demo context by judges, reviewers, and builders evaluating whether a domain-agnostic memory engine can carry durable memory across sessions. The active scenario is a learner working through a difficult topic, but the interface must also make the hidden memory engine legible to an observer.

## Product Purpose

Reverie is a visible memory engine. It extracts durable observations, dreams over session evidence, forgets stale memories, and recalls only the memories that fit a fixed context budget. Success means the app feels like a live AI mind forming and reusing memories, not a generic chatbot with a graph bolted on.

The engine contains zero domain knowledge. Swap one script file and the same engine remembers a customer across support tickets, a patient across visits, an engineer across a codebase. The test suite proves it.

## Brand Personality

Cinematic, intimate, intelligent. The product should feel like a night-sky control room for thought: precise enough to trust, atmospheric enough to remember.

## Anti-references

Avoid run-of-the-mill chatbot layouts, generic SaaS dashboards, plain card grids, gray-on-black developer tools, decorative AI gradients, and visual effects that hide whether the underlying memory system is real.

## Design Principles

Show memory as thought in motion. The primary visual proof should be memories connecting, lighting, fading, and being recalled inside a brain-like constellation.

Make the Session page the stage. The front page should closely follow the supplied 16:9 reference image while preserving real chat, memory, and budget interactions.

Use drama with discipline. Typography, color, glow, and cosmic atmosphere should support the product story without compromising legibility or task flow.

Keep every page in the same world. Dream, Evals, Conductor, Architecture, and navigation should inherit the same editorial transcript, celestial neural, and precision-instrument language.

Prefer real UI over pasted art. Recreate the reference with responsive layout, CSS, canvas, and live app data instead of using the screenshot as a static background.

## Accessibility & Inclusion

Prioritize desktop 16:9 and resizable desktop windows. Maintain readable contrast, visible focus states, keyboard-operable controls, and reduced-motion fallbacks for animated canvas and UI transitions.
